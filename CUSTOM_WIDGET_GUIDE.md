# CustomDnDTaskList ウィジェット設計書

| 項目 | 内容 |
|------|------|
| ドキュメント種別 | 設計書（実装方針・要件定義） |
| 対象ウィジェット | `CustomDnDTaskList`（Mendix Pluggable Web Widget） |
| 想定用途 | CMMS における保全タスクの**実行手順**をリスト表示し、順序をドラッグ＆ドロップで変更して永続化する |

---

## 1. 目的・背景

保全作業では、タスクを**手順どおり**実行する必要がある。本ウィジェットは、Mendix のデータソースとして与えられた **Task の一覧**を、実行手順としてリスト表示する。各行には **タスク名（手順ラベル）**に加え、**タスクの説明欄**（実行内容・注意事項などの補足テキスト）を表示する。各 Task は **SortOrder** で論理順を表し、画面上は **SortOrder 昇順**で表示する。ユーザーが **ドラッグ＆ドロップ**で順序を入れ替えたとき、一覧内の各 Task の **SortOrder を上書きし、コミット**する。行右の **詳細／削除**アイコンから、行ごとのマイクロフロー等を実行できる。

---

## 2. スコープ

### 2.1 本設計で含めるもの

- データソース（List of Tasks）に基づくタスクリストの表示
- **タスクの説明欄**の表示（ウィジェットで Task の説明用属性を指定）
- SortOrder による昇順ソート表示
- DnD による並び替え UI
- 並び替え結果に応じた SortOrder の更新とコミット
- 行単位の **詳細・削除**アクション（Studio でマイクロフロー等を設定）

### 2.2 本設計で明示的に含めない／別途判断するもの

- マイクロフロントやネイティブ（モバイル）対応（現行プロジェクトは **Web** を前提）
- 複数ユーザー同時編集時の競合解決アルゴリズムの詳細（必要なら別紙）
- **写真・画像の表示**（本ウィジェットのスコープ外）

---

## 3. ドメイン・データモデル

### 3.1 エンティティ

保全タスクの 1 行に相当するエンティティ（例: `Tasks`）をデータソースとする。エンティティ名・モジュール名はアプリ定義に従う。

### 3.2 必須属性（本ウィジェットが前提とするもの）

| 属性 | 型（Mendix 上の想定） | 説明 |
|------|----------------------|------|
| SortOrder | Integer（または Long） | 表示・永続化のための並び順。昇順が「手順の前から後」 |
| （説明） | String / Unlimited String 等 | 実行手順の補足。**タスクの説明欄**にバインドする属性。ドメイン上の属性名は任意（例: `Description`, `Instruction`） |

---

## 4. 機能要件

### 4.1 リスト表示

- **入力**: ウィジェットに設定された **List of Tasks**（データソース）
- **表示内容**（最低限）
  - **タスク名（手順ラベル）**: 手順として識別できる主テキスト（`taskNameAttribute` 等で指定）
  - **タスクの説明欄**: 手順の詳細・注意事項などを示す副テキスト（`taskDescriptionAttribute` で指定）。長文の場合は `descriptionMaxLines` で行数制限可
  - **SortOrder 昇順**で並べた行リスト
- **説明が空のとき**: 説明用属性が未入力の Task では、説明欄を非表示にする
- **空データ**: タスクが 0 件のときは空状態を表示

### 4.2 ソート

- 初期表示およびデータ再取得後は、**SortOrder の昇順**でソートして描画する。

### 4.3 ドラッグ＆ドロップ

- ユーザーが行をドラッグし、**別の位置にドロップ**したとき、画面上の順序を入れ替える。
- 入れ替え確定時、対象 Task に **新しい SortOrder**（1 始まりの連番）を割り当て、クライアント API で更新する。

### 4.4 行アクション

- **詳細（行）**: その行の Task を引数に実行するアクション（ページ表示、マイクロフロー等）
- **削除（行）**: 同様に行コンテキストで実行するアクション

## 5. ウィジェットプロパティ設計（`CustomDnDTaskList.xml`）

| グループ | キー | 説明 |
|----------|------|------|
| データソース | `tasks` | 表示・DnD の対象となる Task 一覧 |
| データソース | `sortOrderAttribute` | 並び替え・更新対象の SortOrder |
| データソース | `taskNameAttribute` | 行の主ラベル（任意） |
| データソース | `taskDescriptionAttribute` | 説明欄（任意） |
| 表示 | `descriptionMaxLines` | 説明の最大行数（0 で無制限） |
| 行アクション | `onTaskDetail` | 詳細アイコン押下時（任意） |
| 行アクション | `onTaskDelete` | 削除アイコン押下時（任意） |
| イベント | `onSortOrderChanged` | SortOrder 反映後（任意） |

**注意**: `typings/CustomDnDTaskListProps.d.ts` は XML と同期させる。

---

## 6. 画面・UI 設計（概要）

- 縦リスト（1 Task ＝ 1 行）。
- 各行: **ドラッグハンドル** ＋ **タスク名** ＋ **説明**（任意）＋ **詳細／削除アイコン**（設定時のみ有効化）。

---

## 7. 処理フロー（初期・DnD）

1. `tasks` を取得し、`SortOrder` 昇順で描画する。
2. DnD 確定後、新しい順序で各 Task の `SortOrder` を連番で `setValue` し、`reload` と任意の `onSortOrderChanged` を実行する。

## 9. 受け入れ条件（テスト観点）

- SortOrder 昇順で表示される。
- DnD 後に SortOrder が期待どおり更新され、再表示で順序が維持される。
- 説明欄は仕様どおり表示／非表示になる。
- 詳細・削除アクションは、Studio 設定に従い実行できる（権限・パラメータはアプリ側で保証）。
